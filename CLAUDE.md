# Claude Code instructions

## Git
- Never run git commands (commit, push, add, stash). The developer handles all version control.

## Code quality
- After making any changes, always run:
  1. `ruff check . --fix && ruff format .`
  2. `mypy infra/ track_automl/src/ --ignore-missing-imports`
- Fix all errors before considering a task done.

## Project conventions
- Follow Conventional Commits for suggested commit messages (but do not commit).
- All new code goes in the appropriate track directory: `track_genai/src/` or `track_automl/src/`.
- Shared auth and setup code goes in `infra/`.
- Do not modify `track_automl/telco_churn_pipeline.yaml` directly — it is generated by `pipeline.py`.

## Environment
- Python 3.12, virtual environment at `.venv/`
- GCP project: see `.env` (never commit `.env`)
- Run scripts from the project root, not from inside `src/`
